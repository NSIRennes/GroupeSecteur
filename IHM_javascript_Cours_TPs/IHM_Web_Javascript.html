1. Le DOM et Javascript

Introduction

Le Document Object Model (DOM) est une interface de programmation (appelée aussi API, Application Programming Interface) qui va permettre, via le Javascript, d'accéder au code HTML d'un document. Grâce au DOM, on pourra modifier des éléments HTML (afficher ou masquer un <div> par exemple), en ajouter, en déplacer ou en supprimer.

JavaScript est un langage de programmation orienté objet employé dans les pages web interactives. Il a été créé en 1995 par Brendan Eich . Les premiers navigateurs possédaient des DOM différents ce qui obligeait à coder différemment en fonction du navigateur. Le W3C a permis d'y remédier en publiant les spécifications du DOM : nous nous intéresserons à la dernière version, le DOM 2.

Chaque paire de balises (ouvrante et fermante) de la page HTML est perçue par javascript comme un objet (un élément HTML). Chaque élément HTML possède des propriétés et des méthodes qui ne sont pas forcément les mêmes pour tous. Tous les éléments HTML sont du type Node (nœud) et ont, par conséquent, des propriétés et méthodes communes.

Les objets/éléments

les deux objets de départ :
  • l'objet window : c'est un objet global qui représente la fenêtre du navigateur ; c'est celui auquel on fait référence lors que l'on utilise window.console.log() pour écrire dans la console ;
  • l'objet document : c'est un sous-objet de window qui représente la page Web (la paire de balises HTML)

Accéder aux élements

Trois méthodes principales :
  • getElementById() : cette méthode permet d'accéder à un élément en connaissant son identifiant (id) ; cet identifiant doit être unique dans la page ;
  • getElementsByTagName() : cette méthode permet de récupérer dans un tableau tous les éléments d'une famille d'éléments, par exemple tous les div
  • getElementsByName()  : cette méthode permet de récupérer dans un tableau tous les éléments qui possèdent le même nom (attribut name) ; à réserver aux éléments des formulaires
Deux méthodes récentes pas forcément supportées par tous les navigateurs : elles s'utilisent sur les sélecteurs CSS
  • querySelector() : renvoie le premier élément trouvé correspondant au sélecteur CSS
  • querySelectorAll() renvoie sous forme de tableau tous les éléments

Editer les éléments

  • getAttribute() : permet de récupérer l'attribut spécifié en argument
  • setAttribute() : permet de le modifier
  • innerHTML : permet de récupérer et/ou modifier le code HTML d'un élément sous forme de texte

Code des exemples :

Le code correspondant au changement de texte du titre est le suivant : 
var titre=document.getElementById('intro');
window.alert("le <h3 > de haut de page contient le texte : "+titre.innerHTML+" . On va le modifier.");
titre.innerHTML="Un peu de vocabulaire";
Le code correspondant au changement de couleur du bloc est le suivant : 
var mondiv=document.getElementById('introduction');
window.alert(" le premier paragraphe est sur fond jaune ; on va en modifier la couleur.");
mondiv.className="couleur";
Ici, on a joué sur le fait que le code attend que la fenêtre d'alerte ait été fermée pour s'exécuter. Ce n'est pas la bonne stratégie pour rendre une page interactive : il est préférable de se servir d'événements.(voir paragraphe suivant)
